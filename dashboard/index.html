Plug-and-Play Ecosystem Architecture
How modules connect
📋 Module Manifest SDK
Every module declares a ModuleManifest with name, version, capabilities, required bindings (D1/KV/R2/AI/DO/Queue/Vectorize), routes, events, config schema, and minimum plan tier. The SDK validates at construction time.
🔄 Lifecycle Hooks 7 hooks
install → enable → handleRequest → handleEvent → disable → upgrade → uninstall. Plus healthCheck for monitoring. All hooks are optional with graceful no-op defaults.
🌐 Route Matching Glob
Modules declare routes with glob patterns: /api/users (exact), /api/users/* (wildcard), /api/** (catch-all). The SDK matches incoming requests and dispatches to the right handler.
🔐 Plan-Based Access 4 tiers
Modules specify minimum plan: free → starter → pro → enterprise. The SDK's meetsMinimumPlan() gates feature access per tenant subscription level.
📡 Event-Driven 30+ events
Modules subscribe to platform events via patterns. When a tenant.created or user.login fires, all matching modules receive the event asynchronously through the Event Bus.
🔗 Dependency Resolution DAG
The Registry Worker resolves module dependencies recursively with cycle detection. Install order is topologically sorted. Missing or circular dependencies are caught and reported.
